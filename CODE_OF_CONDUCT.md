# Code of Conduct

## Our Standard

Project discussions should stay professional, concrete, and focused on the
technical work. Contributors are expected to:

- be respectful of differing viewpoints,
- give and receive feedback constructively,
- avoid harassment, personal attacks, and discriminatory language,
- keep security-sensitive information out of public threads,
- assume good intent while still requiring evidence for technical claims.

## Unacceptable Behavior

Unacceptable behavior includes:

- harassment or intimidation,
- discriminatory comments,
- publishing private information without permission,
- sharing exploit details against third-party systems,
- repeatedly derailing technical discussions,
- using the project to target systems without authorization.

## Enforcement

Maintainers may remove comments, close issues, restrict participation, or report
abuse to GitHub when behavior violates this code of conduct.

For security-sensitive concerns, use the process in `SECURITY.md`.
